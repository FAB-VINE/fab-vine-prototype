# 📦 Project Overview

## Roles & Responsibilities
- **Artists:** Work in `BUILD-FILES/3d-prints` and `SIMULATION`.
- **Technicians:** Follow guides in `MANUALS` and use `LAUNCH-TOOLS`.
- **Developers:** Code in `NODE-LOGIC` and `BRAIN-SERVER`.

## The Sprints
1. **Seed:** Proof of Concept (Sprint 1)
2. **Branch:** Snap-Mechanism (Sprint 2)
3. **Trunk:** Vertical Stack (Sprint 3)
4. **Canopy:** Full Tree (Sprint 4)
5. **Forest:** 50-Node Scale (Sprint 5)

## The folders
| Directory Name | Who Uses It? | What They See (Mental Model) |
| :--- | :--- | :--- |
| **`MANUALS`** | Everyone | "The instruction book." No jargon. Just guides. |
| **`BUILD-FILES`** | Artists, Technicians | "Where the physical stuff lives." Clear distinction between 3D prints and PCBs. |
| **`NODE-LOGIC`** | Developers, Techs | "The code inside the boxes." Split into "Sense" (Eyes) and "Light" (Voice) is intuitive. |
| **`BRAIN-SERVER`** | Devs, Data Scientists | "The central computer." Split into "Core" (Real-time) and "Data-Hub" (Analytics). |
| **`SIMULATION`** | Artists, Designers | "The virtual sandbox." Test ideas before building. |
| **`LAUNCH-TOOLS`** | Technicians, Makers | "The magic buttons." Scripts to flash, update, and test hardware. |
