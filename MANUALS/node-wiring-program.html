Node V1 / Wiring + Firmware

How the current Fab Vine node is connected and programmed.

This manual explains the current six-face prototype: how the XIAO ESP32-C6 is wired, how each face detects a neighbor, how the OLED and LEDs connect, and what the program does.

Resumen rapido

El nodo funciona como un ser social. Si esta solo, respira en morado. Si una o mas caras detectan vecino, el rojo desaparece y el nodo se siente mas seguro: saluda en la pantalla OLED y respira en azul verdoso. En Sprint 2, los modulos tambien pueden elegir un lider aleatorio usando una linea de comunicacion en D8.

  • Microcontrolador: Seeed Studio XIAO ESP32-C6.
  • Pantalla: OLED SSD1306 128x64 por I2C.
  • Luces: 40 LEDs WS2812B / NeoPixel en una tira zig zag.
  • Vecinos: 6 caras, cada una con un pin de deteccion.
  • Conteo: el programa cuenta de 0 a 6 caras ocupadas.

Face detect wiring

Cada cara usa una entrada digital con INPUT_PULLUP. Eso significa que el pin normalmente lee HIGH. Cuando una cara se conecta fisicamente con un vecino, esa cara debe conectar su pin DETECT a GND; entonces el programa lee LOW y cuenta esa cara como ocupada.

Leader election wiring

Para que los modulos elijan lider de forma democratica, necesitan una linea extra para comunicarse. En esta version usamos D8 como DATA / SIGNAL. Los dos modulos conectan D8 con D8 y tambien comparten GND.

  • D8 trabaja a 3.3V; no conectar a 5V.
  • La linea se usa como bus open-drain: los modulos solo jalan la linea a LOW o la sueltan.
  • El programa activa el pull-up interno de D8, asi que se puede probar sin resistencia externa.
  • Una resistencia 10k de D8 a 3V3 sigue siendo recomendable para una version mas estable.
  • Cada modulo espera un tiempo aleatorio; el primero que toma la linea se vuelve lider.
  • El lider mantiene D8 en LOW como senal de presencia, y el otro modulo se queda como seguidor.

What the program does

  • Reads the six face pins continuously.
  • Uses debounce so a noisy contact does not count many times.
  • Stores which faces are connected in faceConnected[6].
  • Calculates neighborCount from 0 to 6.
  • If neighborCount == 0, the node is alone and breathes purple.
  • If neighborCount changes from 0 to 1 or more, the OLED performs a yes/nod greeting.
  • When connected, both modules start a random wait over D8.
  • The first module to claim D8 becomes leader and holds the line LOW.
  • The leader shows confident eyebrows on the OLED and breathes stronger blue.
  • The follower keeps the normal/wink OLED behavior and breathes blue-green.
  • If the count returns to 0, the OLED performs a no/head-shake and the module returns to purple.
Program state logic:

0 neighbors
  -> NODE_IDLE
  -> purple breathing

0 to 1+ neighbors
  -> NEIGHBOR_CONNECTED
  -> OLED yes gesture
  -> connected face LEDs green/blue

1+ neighbors staying connected
  -> CONNECTED_IDLE
  -> random leader claim over D8
  -> leader: eyebrows + stronger blue
  -> follower: calm blue-green breathing state

1+ to 0 neighbors
  -> NEIGHBOR_REMOVED
  -> OLED no gesture
  -> return to purple idle
