Node V1 / Wiring + Firmware

How the current Fab Vine node is connected and programmed.

This manual explains the current six-face prototype: how the XIAO ESP32-C6 is wired, how each face detects a neighbor, how the OLED and LEDs connect, and what the program does.

Resumen rapido

El nodo funciona como un ser social. Si esta solo, respira en morado. Si una o mas caras detectan vecino, el rojo desaparece y el nodo se siente mas seguro: saluda en la pantalla OLED y enciende posiciones de la tira LED en verde/azul.

  • Microcontrolador: Seeed Studio XIAO ESP32-C6.
  • Pantalla: OLED SSD1306 128x64 por I2C.
  • Luces: 40 LEDs WS2812B / NeoPixel en una tira zig zag.
  • Vecinos: 6 caras, cada una con un pin de deteccion.
  • Conteo: el programa cuenta de 0 a 6 caras ocupadas.

Esquema general

                    XIAO ESP32-C6
                 +----------------+
Face 1 DETECT -> | D0             |
Face 2 DETECT -> | D1             |
Face 3 DETECT -> | D2             |
Face 4 DETECT -> | D3             |
OLED SDA      -> | D4             |
OLED SCL      -> | D5             |
Face 5 DETECT -> | D6             |
Face 6 DETECT -> | D7             |
LED DATA      -> | D10            |
GND -----------> | GND            |
3V3 -----------> | 3V3            |
                 +----------------+

OLED SSD1306:
VCC -> 3V3
GND -> GND
SDA -> D4
SCL -> D5

LED strip WS2812B / NeoPixel:
5V  -> external 5V power supply
GND -> external power GND + XIAO GND
DIN -> D10

Important:
Do not power the LED strip from the pogo / face detect pins.

Face detect wiring

Cada cara usa una entrada digital con INPUT_PULLUP. Eso significa que el pin normalmente lee HIGH. Cuando una cara se conecta fisicamente con un vecino, esa cara debe conectar su pin DETECT a GND; entonces el programa lee LOW y cuenta esa cara como ocupada.

LED zig zag map

La tira tiene 40 LEDs, pensados como 4 lineas fisicas de 10 LEDs. El programa no divide la tira en sectores continuos. En cambio, enciende posiciones repetidas del zig zag.

What the program does

  • Reads the six face pins continuously.
  • Uses debounce so a noisy contact does not count many times.
  • Stores which faces are connected in faceConnected[6].
  • Calculates neighborCount from 0 to 6.
  • If neighborCount == 0, the node is alone and breathes purple.
  • If neighborCount changes from 0 to 1 or more, the OLED performs a yes/nod greeting.
  • If one or more faces stay connected, the strip breathes in blue-green.
  • If the count returns to 0, the OLED performs a no/head-shake and the module returns to purple.
Program state logic:

0 neighbors
  -> NODE_IDLE
  -> purple breathing

0 to 1+ neighbors
  -> NEIGHBOR_CONNECTED
  -> OLED yes gesture
  -> connected face LEDs green/blue

1+ neighbors staying connected
  -> CONNECTED_IDLE
  -> calm blue-green breathing state

1+ to 0 neighbors
  -> NEIGHBOR_REMOVED
  -> OLED no gesture
  -> return to purple idle

Upload notes

  • Board: Seeed Studio XIAO ESP32-C6.
  • FQBN: esp32:esp32:XIAO_ESP32C6.
  • Tested upload port: COM3.
  • Required libraries: Adafruit GFX Library, Adafruit SSD1306, Adafruit NeoPixel.
  • Main firmware file: NODE-LOGIC/fab-vine-node/V1/firmware/fab-vine-node-v1/fab-vine-node-v1.ino.
