How New Manuals Should Fit

Add specific documentation near the files it explains, then expose the human-friendly version here. This keeps technical notes close to the work while giving contributors one clear place to explore.

  • Use MANUALS/index.html as the main exploration menu.
  • Use area cards for broad concepts: physical node, node logic, brain server, simulation, launch tools, and community.
  • Create specific HTML manuals in MANUALS/ when they should be readable by non-technical contributors.
  • Keep detailed Markdown near firmware, hardware, or server code when it helps development.
  • Use English as the primary language and Spanish as the secondary support language.
