Programa V3

El programa V3 convierte el nodo en una unidad social que transmite estado, escucha a sus seis caras y actualiza la interfaz visual sin bloquear el loop principal. La comunicacion usa paquetes con marcador de inicio, payload, checksum y marcador de fin. 

Loop principal V3:

1. Parpadea el LED integrado como senal de vida.
2. Lee el boton para reiniciar el estado local.
3. Cada segundo envia PacketData por D6 / Serial1.
4. Escucha los seis RX con SoftwareSerial.
5. Marca vecinos activos cuando llegan paquetes validos.
6. Expira vecinos si pasan 3 segundos sin paquete.
7. Actualiza OLED y LEDs en intervalos independientes.
8. Cada 2 segundos recalcula selfState.

Interfaz OLED y LEDs

La interfaz ya no muestra solo numeros de debug. V3 usa ojos expresivos y pequenos marcadores de cara: los marcadores se mantienen visibles mientras una cara espera vecino y desaparecen cuando esa cara esta conectada.

  • EyesNormal: mirada estable para estado base.
  • EyesBlink: parpadeo cuando el nodo esta solo.
  • EyesWink: gesto corto cuando hay pocos vecinos.
  • EyesLookLeft / EyesLookRight: mirada lateral cuando hay tres o mas conexiones.
  • LEDs solos: respiracion morada.
  • LEDs conectados: respiracion azul/verde sin canal rojo.
  • LEDs con tres o mas vecinos: azul mas intenso y periodo un poco mas rapido.

Logica de dado

El valor selfState funciona como una semilla o dado interno. Cada nodo arranca con un valor pseudoaleatorio entre 5 y 254. Cuando detecta vecinos, promedia los estados recibidos y mueve su propio estado hacia ese objetivo usando interpolacion suave. 

Al iniciar:
selfState = random(5, 254)

Cada ciclo de computo:
targetValue = promedio de estados vecinos activos
selfState = lerp(selfState, targetValue, 0.187)
blinkInterval = selfState * 3

Esto hace que el comportamiento no sea un simple encendido/apagado. La red tiende a sincronizarse gradualmente, como si cada pieza negociara su estado con las piezas alrededor.

Pruebas pendientes

  • Probar dos placas reales conectadas cara a cara: TX comun de una placa debe llegar al RX de la otra.
  • Confirmar que SoftwareSerial puede escuchar las seis caras con la estabilidad necesaria.
  • Validar si el paquete actual necesita incluir mas informacion de posicion para mallas grandes.
  • Revisar alimentacion de LED strip antes de usar muchos nodos juntos.
  • Confirmar si el boton en A0 seguira en la placa final o quedara solo para pruebas.
