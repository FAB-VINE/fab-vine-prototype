Node Logic / V1

V1 is the first unified Fab Vine node firmware.

All nodes are currently expected to run the same base firmware. Differences should be handled through configuration or future versions, not separate sketches for each node.

What V1 Contains

  • OLED expression display behavior.
  • Places for sensor logic, LED behavior, and network communication.
  • A local preview tool for OLED expressions.
  • An Arduino sketch tested on XIAO ESP32-C6.

Current MVP

The current firmware is an offline six-face physical-neighbor test. Each face has a DETECT line using an internal pullup: no neighbor reads HIGH, and a connected neighbor pulls that face to GND so it reads LOW. The firmware counts occupied faces from 0 to 6.

When the module is alone, the 40 WS2812B LEDs breathe in purple to suggest slight social anxiety without feeling like an alarm. When contact is detected, the red disappears: the OLED performs a soft greeting expression and the LEDs pulse green/blue to show a safer connected state. The strip is mapped as a zig zag, so a physical position such as 1 lights LEDs 1, 11, 21, and 31. Do not carry main LED power through pogo pins; use face connectors only for detect or signal behavior.

Zig Zag LED Map

The strip is treated as four physical lines of ten LEDs. A face does not own a continuous sector; it lights a repeated physical position across the zig zag.

  • Position 1 lights physical LEDs 1, 11, 21, and 31.
  • Position 3 lights physical LEDs 3, 13, 23, and 33.
  • Position 10 lights physical LEDs 10, 20, 30, and 40.

The current default map is: Face 1 = position 1, Face 2 = position 3, Face 3 = position 5, Face 4 = position 6, Face 5 = position 8, Face 6 = position 10.
